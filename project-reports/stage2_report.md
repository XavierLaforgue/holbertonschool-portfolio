# Portfolio project - Stage 2: Project planning
## Timeline
The portfolio project of the Holberton School has a duration of 14 weeks.
Below is the expected timeline for the project's multiple stages. 
| Stage | Phase | Week | Dates | Status |
|-------|-------|------|-------|--------|
| 1 | Team formation, brainstorming, and MVP | 1 | 08/12/25 -- 15/12/25 | Completed |
| 2 | Project planning | 2 | 16/12/25 -- 19/12/25 | In progress |
| 3 | Technical documentation | 3 | 05/01/26 -- 12/01/26 | On time |
| 4 | MVP development and execution | 4 -- 11 | 13/01/26 -- 06/03/26 | On time |
| 5 | Project closure and landing page | 12 -- 13 | 09/03/26 -- 20/03/26 | On time |
## Milestones and deliverables
Each project stage has associated milestones and deliverables.
These may be described as follows.
1. **Week 1**: Team formation, brainstorming, and MVP (Completed)
   * Brainstorm ideas.
     * *Milestone*: List topics and non-evaluated ideas (problems, trends, and solutions).
   * Mindmapping.
     * *Milestone*: Visualize related ideas and group them in themes.
   * Idea evaluation.
     * *Milestone*: Short-list ideas, describe and evaluate/rank them.
   * Decision.
     * *Milestone*: Choose an idea.
   * Refinement.
     * *Milestone*: Detail chosen idea and define a few features and objectives of the minimal viable product (MVP).
   * Risk assessment and mitigation strategies.
   * Establish project scope.
     * *Deliverable* (week 1): stage 1 report documenting the whole process from ideation to decision, MVP refinement, and definition of the project's scope.
2. **Week 2**: Project planning (In progress)
   * Set timeline.
     * *Milestone*: Develop high-level plan of the project's phases.
   * Detail milestones and deliverables.
   * Design roadmap.
     * *Milestone*: Make Gantt chart of the project roadmap.
     * *Deliverable* (week 2): stage 2 report documenting the plan and roadmap of the project from ideation to project closure.
3. **Weeks 3**: Technical documentation (On time)
	* Write user stories.
    	* *Milestone*: Define the actions that users of the MVP will will be able to perform.
	* Design mockups.
    	* *Milestone*: Design a UI that covers the allows the user to access the main functionalities of the MVP.
  * Create diagrams and plan system architecture.
		* *Milestone*: architecture, class, sequence, UI components, and
		  entity-relationship diagrams.
	* Provide rationale for selected technologies.
		* *Deliverable* (week 3): Stage 3 report.
4. **Weeks 4 -- 11**: MVP development and execution (On time)
  * Plan and define sprints.
    * *Milestone*:  Plan development tasks and divide them into sprints.
    * *Deliverable* (week 11): Gantt chart with planned sprints and real sprints after applied corrective measures. 
  * Execute development tasks while tracking bugs.
    * *Deliverable* (week 11): Codebase of completed MVP with clear track of bug detection and correction.
  * Monitor project and adjust
  * Sprint reviews and retrospectives
    * *Milestone*: Evaluate finished sprint and adjust sprint planning or execution method for the next ones.
  * Final integration and QA testing.
    * *Milestone*: Have a working test suite.
  * Deliver production environment.
    * *Milestone*: Pass the MVP from development to production environment.
    * *Deliverable* (week 11): Production-ready Docker image.
  * Write documentation.
    * *Deliverable* (week 11): Project documentation.
5. **Weeks 12 -- 13**: Project closure and landing page (On time)
  * Document results and lessons learned.
  * Prepare project presentation.
  * Perform team retrospective.
  * Produce a landing page for the MVP.
    * *Milestone*: Write an intro section.
    * *Milestone*: Write a feature section.
    * *Milestone*: write an about section.
    * *Deliverable* (week 13): Landing page (github pages).
  * Deliver project presentation.
    * *Milestone*: Present the project to a jury of experts.
    * *Deliverable* (week 13): Project presentation.
    * *Deliverable* (week 13): Final report.
